Identifier
  1. Let ref be the result of performing ResolveBinding (StringValue (Identifier)).
  2. Return ref.
yield
  1. Assert : The source code matching this production is not strict code .
  2. Let ref be the result of performing ResolveBinding ("yield").
  3. Return ref.

The result of evaluating an IdentifierReference is always a value of type Reference .

In non-strict code , the keyword yield may be used as an identifier. Evaluating the IdentifierReference production resolved the binding of yield as if it was an Identifier. The Early Error restriction ensures that such an evaluation only can occur for non-strict code . See 13.2.1 for the handling of yield in binding creation contexts.
